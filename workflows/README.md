# Workflows

Markdown SOPs that define how to accomplish specific objectives using available tools.

## Structure of a Workflow

Each workflow file should contain:

- **Objective** — What this workflow accomplishes
- **Inputs** — What's needed before starting
- **Steps** — Ordered sequence of tool calls and decisions
- **Outputs** — What success looks like (and where it goes)
- **Edge Cases** — Known failure modes and how to handle them
- **Notes** — Lessons learned, rate limits, quirks discovered over time

## Naming Convention

`workflows/verb_noun.md` — e.g., `scrape_website.md`, `generate_report.md`, `sync_sheets.md`
